# SkillRL: Evolving Agents via Recursive Skill-Augmented Reinforcement Learning

**Authors:** Peng Xia, Jianwen Chen, Han Wang, Jiaqi Liu, Kaide Zeng, Yu Wang, Siwei Han, Yiyang Zhou, Xujiang Zhao, Haifeng Chen, Zeyu Zheng, Cihang Xie, Huaxiu Yao  ·  **Year:** 2026  ·  **Source:** https://arxiv.org/abs/2602.08234

## One-line
An RL framework that distills agent trajectories into a hierarchical skill library which co-evolves with the policy via GRPO, replacing raw trajectory memory with reusable, abstracted "skills."

## Key claim
Distilling experience into a hierarchical skill library (general + task-specific) and recursively evolving it during RL yields 15.3% average improvement over memory-augmented RL baselines, with 10–20× token compression vs storing raw trajectories.

## Method
Three-stage pipeline: (1) a teacher model $M_T$ distills both successful and failed rollouts into compact skills (success → strategic patterns; failure → counterfactual lessons); (2) skills are organized as $\text{SkillBank} = S_g \cup \bigcup_k S_k$ with general skills always loaded and task-specific skills retrieved by embedding similarity (top-$K$, $K=6$); (3) cold-start SFT teaches the base model to use skills, then GRPO trains the policy while validation-epoch failures (categories with success $< \delta = 0.4$) trigger new-skill synthesis $S_{\text{new}} = M_T(T_{\text{val}}^-, \text{SkillBank})$.

## Result
On Qwen2.5-7B-Instruct: ALFWorld 89.9% (vs GRPO 77.6%, GPT-4o 48.0%, Gemini-2.5-Pro 60.3%); WebShop 72.7% success / 85.2 score (vs GRPO 66.1%); search-QA average 47.1% (vs Search-R1 38.5%, EvolveR 43.1%). Cool/Pick2 subtasks of ALFWorld show +23% and +22.8% absolute over GRPO, indicating skill priors help most in sparse-reward / compositional regimes.

## Why it matters here
General background; no direct arena tie. Methodologically relevant to the einstein self-improvement loop — SkillBank's split between "general skills" (always-loaded heuristics) and "task-specific skills" (similarity-retrieved per task) mirrors the wiki's `concepts/` vs `problems/` split, and the failure-to-skill distillation echoes the `failure-is-a-finding` rule. Suggests a structural design choice: storing principles + when-to-apply rather than raw trajectories.

## Open questions / connections
- Does recursive skill evolution converge, or does SkillBank grow unboundedly across many task categories? Paper caps new-skills-per-evolution at 3 but no long-horizon stability analysis.
- Skills are author-generated by a strong teacher (o3); how does quality degrade when teacher ≈ student? Relevant to self-bootstrapping wiki agents.
- Connection to ExpeL, Reflexion, Voyager, MemRL, EvolveR — extends the "memory for agents" thread by adding *abstraction* (skills) on top of compression.
- Open: optimal similarity threshold $\delta$, retrieval $K$, and whether hierarchical skill structure beats flat for math/optimization domains (not tested here — only ALFWorld/WebShop/QA).
