1. **What math concepts does this problem invoke?**
2. **How rich is it mathematically?** (1–3 — pure polish vs deep theory)

4. **What's the learning ROI?** (S/A/B/C tier — drives priority for the self-improvement cycle)

## Tier definitions

- **S — Rich + low coverage** — highest learning ROI. Conceptually deep; current wiki under-explains the underlying math. Author concepts/techniques pages here first.
- **A — Rich + partial coverage** — high ROI; deepen existing pages and cross-link.
- **B — Well-covered or moderate richness** — good for confirmation; not the priority for new wiki pages.
- **C — Shallow / polish-only** — skip for wisdom mining. Will get a thin `wiki/problems/<id>.md` index page only.

## Synthesis

### Top 5 concepts that span 3+ problems (cross-problem reuse)

1. **Equioscillation + larger-n escape** (P1, P2, P3, P4) — Discretized-function problems with piecewise-constant SOTA converge to Chebyshev equioscillation basins. Block-repeat upsample + tiny perturbation + smooth-max L-BFGS with β-cascade unlocks new DOF. *Reusable check:* if 3+ agents byte-identical, test larger-n escape before declaring frozen.
2. **Float64-ceiling problems** (P11, P14, P17b) — Contact-graph-locked basins where all top agents converge within 1–2 ulps. mpmath 80-digit verification confirms basin's true ceiling. *Reusable:* compute basin ceiling via mpmath KKT before launching ulp lottery.
3. **Harmonic analysis / Fourier-dual reformulation** (P1, P2, P3, P4) — Autocorrelation/overlap problems are fundamentally Fourier-dual. Equioscillation in primal ↔ Chebyshev extremizers in dual. Remez exchange is canonical diagnostic. *Reusable:* Hilbert space inner product + Hardy-Hilbert + Remez exchange test.
4. **Basin rigidity + contact-graph determination** (P5, P6, P11, P14, P16) — Sphere/geometric packing problems are over-constrained: active constraint count often exceeds DOF (P5: 30 vs 28). Multistart and perturbation fail. *Reusable:* count active constraints vs DOF before optimization; if over-constrained, diagnose rigidity and stop wasting compute.
5. **Parameterization selection > optimizer intensity** (P2, P3, P9, P13) — v² vs exp(v), gap-space vs position-space, per-region sigmoid vs unconstrained. Breakthrough often free; intensity escalation costs GPU. *Reusable:* before scaling compute, test ≥2 structurally different parameterizations.

### Top 3 techniques that span 3+ problems

1. **SLSQP polish from warm-start** (P5, P11, P14, P17b, P18) — Smooth NLP reformulation + affine gauge + tight tolerance (ftol=1e-20) → basin float64 floor in 3–5 iterations. Negligible cost.
2. **Parallel tempering SA with contribution-weighted sampling** (P6, P10, P22) — 8–64 replicas, geometric T ladder, periodic Metropolis exchange, perturbation ∝ vector contribution. 730× speedup vs greedy on P6. GPU-accelerated.
3. **Multi-start with diverse seed strategies** (P13, P14, P16, P17a) — Random / grid / lattice / symmetric / near-leader / near-runner-up × 1000+ trials. Discovers 3–10 distinct basins; enables rank-#2 fallback.
